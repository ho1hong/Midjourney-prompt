# User Aesthetic — Learned Patterns（学习层）

> 本文件记录从用户**持续提供的满意图**中归纳的**共性**，用于**后续 prompt 迭代与变体**时迁移，**不是**全局默认某一种画风。  
> **题材模板**（如汽车大片）只在用户当条需求匹配时使用；此处写的是「你为什么觉得这批图对味」的可抽象结构。

---

## 审美母语（Top Principle — 一切判断的根）

> 用户自己提炼（2026-04-21），优先级 **高于** 下面任何数据集、注册、摄影师参照。Agent 做任何风格判断、取舍、迭代前，先在心里对齐这一节。

**一句话定义**  
**「不是在拍车，而是在为车建立世界观。」**  
以**真实工业质感**为根，以**车型人格与品牌语义**为核心，通过**场景、光线、色彩、构图、镜头系统**的精确控制，为汽车建立一个**克制、可信、可系列化**的高端视觉世界。

**属性定位**：不是「摄影作品集审美」，是 **汽车品牌资产化影像审美（brand-asset imagery）**——单张要成立，一组更要成立。

**审美母语（8 词）**  
`真实` · `克制` · `秩序` · `人格` · `空间` · `余韵` · `系统` · `品牌化`

**视觉化落点（8 项，写 prompt 时的可执行映射）**

| 母语 | 落到英文 prompt 的方向 |
|------|------------------------|
| 真实车感 | `photorealistic industrial finish, readable body panels and keyline, accurate paint reflections, factory-correct proportions, no CG plastic look` |
| 克制氛围 | `restrained editorial mood, quiet luxury, minimal VFX, subtle contrast, no blockbuster grade` |
| 规整构图 | `deliberate framing, balanced negative space, clean horizon, purposeful camera placement` |
| 场景共生 | `environment and vehicle share one palette and light logic, location reads as intentional, not backdrop` |
| 色彩呼应 | `limited cohesive palette, paint color responds to environment (metal↔stone, champagne↔pine, silver↔mist), no rainbow clutter` |
| 镜头分工 | 一组多图时分 `hero wide / tracking three-quarter / detail macro / interior POV / KV symmetry`，**一张一职**，不重复 |
| 叙事留白 | `one quiet focal event, open negative space, no prop overload, no busy storytelling` |
| 系列统一 | 同项目跨张 `consistent color grade, consistent time-of-day family, consistent lens family, consistent compositional grammar` |

**系统级关切（写组图/系列时额外检查）**  
- 同一项目里 **不同场景** 如何统一（调色族、光族、镜头族 三族对齐）  
- 不同角度如何 **避免重复**（hero / tracking / detail / POV / KV 各司其职）  
- 不同时间段如何 **形成呼应**（晨雾 → 金时刻 → 蓝调，走一条色温弧，不乱跳）  
- 不同车色如何 **建立对应环境**（银灰↔雾林 / 古铜↔大地 / 钴蓝↔盐白 / 暖枪灰↔混凝土）  
- **轿车 vs 猎装/旅行** 版：同源（同家族视觉语法）又区分**人格**（轿车更都市秩序，猎装更户外叙事）  
- 场景图 / 功能图 / KV / 路跑图要进入 **同一视觉系统**（哪怕任务不同，一眼能看出是一家）

**反审美（绝对排斥，写进 prompt 的反向约束）**

`fake CG look`、`video-game render`、`cheap cyberpunk neon overload`、`unjustified extreme wide-angle distortion`、`over-the-top low-angle hero worship`、`shock-value-only composition`、`landscape overpowering the car`、`element overload`、`disordered narrative clutter`、`oversized prop staging`、`no lens division across a set`、`style that kills model credibility`、`generic cool shot without brand character`

写成英文 `--no` 或自然语言反向约束时常用：  
`no plastic CG finish, no video-game render, no cyberpunk neon clutter, no unjustified ultra-wide distortion, no over-dramatic worm's-eye, no landscape dwarfing the car, no prop overload, no teal-orange blockbuster grade, no generic cool shot without brand identity`

**四条取舍底线（决策优先序）**  
1. **宁可克制，也不要失控**  
2. **宁可真实，也不要假炫**  
3. **宁可系列统一，也不要单张过猛**  
4. **宁可设计过的安静，也不要廉价的热闹**

## 使用约定（给 Agent）

1. **所有 prompt 先对齐上面的「审美母语」**：真实车感、克制氛围、规整构图、场景共生、色彩呼应、系列统一（一张任务也要隐含「可被扩成一组」的基因）。  
2. **新增满意图**：先找**跨图重复**的结构，再写进「可迁移共性」；仅属于某一类主体的（如轮毂、车型）放进「分题材观察」。  
3. **写 prompt / 迭代**：**优先**用户当条指令 → 再选 **1–3 条**「可迁移共性」融入（不冲突时）→ 最后才考虑 `--p`。  
4. **组图 / 系列任务**：按「镜头分工 × 色温弧 × 色彩呼应」建立矩阵，明确每张在系统里的职位（hero / tracking / detail / POV / KV），**不要**出同职位重复。  
5. **禁止**：把当前数据集里出现过的主体（如某类车）当默认，除非用户明说。  
6. **矛盾时**：用户明确硬指令 > 当条任务目标 > 审美母语（默认审美与防跑偏 guardrail） > 本文件共性 > 数据集观察。  
7. **反审美清单**是**硬约束**：触发其中任一项即重写，不要心疼描述句。

## 可迁移共性（跨题材，优先维护）

以下为从用户**多批**参考中抽象出的、**不绑定具体主体**的倾向。新图若反复印证则加强表述；若出现**不同「光感注册」**则标为分支，迭代时按用户当条意图选用，**不**把某一注册当全局默认。

| 维度 | 共性描述 |
|------|----------|
| 商业气质 | **Editorial / commercial premium**：像品牌片或杂志片，完成度高；**静奢 Quiet luxury**，不廉价堆特效。 |
| 对比与光（多注册） | **注册 A（城市动感）**：硬光、切割影、跟拍时主体锐、环境动态模糊（见数据集 A）。**注册 B（生活/自然）**：**黄金时刻**软而方向明确的光、长而柔的阴影；或 **阴天/薄暮** 的柔光散射。**注册 C（氛围自然）**：林间 **体积光 / 晨雾 / 丁达尔（Tyndall）**、**斑驳树影 dappled sunlight**，冷暖雾区对比；常与 **银灰金属漆 + 深绿林相** 同屏（见 B/C 森林批次）。迭代时**只选一条主注册**；同一语义内可写「金时刻 + 雾」但避免与 A 的硬城市场景混写。 |
| 色彩 | **低饱和主盘** + 克制的冷暖或大地色：**香槟 / 古铜 / 鼠尾草绿 / 暖枪灰** 等金属漆在 B 批反复出现；A 批偏冷银、白、冰蓝；**C 批**大量 **冷银灰旅行车 + 森林绿 + 金光束**。**排斥**廉价「青橙大片预设」感（除非用户要）。 |
| 清晰区与层次 | **主次分工**：该锐的（车身关键面、人物活动区）清晰；背景可退后。**前景层**常用虚化的草叶、树干、狗、虚焦的另一辆车、吊床边缘、尾灯光斑等做 **框景与纵深**（多图重复，已升为共性）。 |
| 空间与机位 | **非平淡平视**：低机位压车、略高跟踪、广角环境叙事、对称正后方等；生活场景 **眼平广角**；**车内叙事**可出现 **天窗框景 / 车内仰拍 POV**（见数据集 C），与车外大片二选一或分镜使用。 |
| 人物与生活 | 若出现人：**自然抓拍感**（徒步、家庭动线、情侣与相机），户外 **Gorpcore-lite / 静奢度假**；**动物极前景虚化**（狗贴镜）作 **即兴感**，不抢车身可读性。**不**必每张都带人。 |
| 动态与静止 | 同一大审美下可分支：**静止露营陈设** vs **林道侧视 + 轮辐/环境动感模糊**；依当条指令选一条，避免「露营桌 + 高速模糊」无逻辑叠用。 |
| 质感底线 | 拒绝 **塑料磨皮、蜡感高光、假 HDR 光晕**；保留 **自然肤色与织物**（有人时）、车漆 **可读反射与体积**。 |
| 商业 × 艺术（注册 D，可选） | **产品/品牌可读性**（车漆、 keyline、灯带、材质）与 **非目录式镜头** 并存：允许 **强烈 Dutch tilt**、**广角夸张的纵深**、**抽象布景**、**人物时装化前景虚化**、**反射/眩光作为叙事**。**不**把「每张都歪地平线」当默认——仅当用户要「广告片但更艺术」时启用（详见下文摄影师参照）。 |
| 地面色统一法（跨题材可迁移） | **给场地铺一块品牌色绒毯 / 染色地坪 / 油漆色块**，把真实建筑/自然场收敛为**品牌色宇宙**；这一手法可做 **双车同项目** 的桥接（不同车色通过同一张「色地」进入同一色系），也可用于单车 KV 的**舞台化**。提示词方向：`campaign-coded colored ground plane (lavender carpet / painted tarmac / pigment runway) binding vehicle hue to environment, stage-like abstraction, not a normal road`。高饱和地面与低饱和天空/建筑对撞时，**天/墙必须留给负空间**，不要满屏炫色。 |
| 灯签名作肖像（跨题材可迁移） | **头灯 DRL / 贯穿尾灯** 作为一张独立镜头职位——**近距微距 + 低环境光 + 让灯本身发光当造型**，画面里可出现半张人脸被灯打亮，但车身其它部分全部暗下去。这是「把灯签名当品牌 logo 拍」。提示词方向：`close-up light-signature portrait, DRL bar or full-width tail-lamp as the graphic hero, near-black ambient, subject's half-face rim-lit by the lamp only, lamp reads as sculpture`。**只**在组图里留 1–2 个这类职位，别重复。 |
| 反射复写（跨题材可迁移） | **玻璃幕 / 水面 / 漆面** 把同一画面拍两次，常用来**把封闭室内与户外车并置**、或**把人物倒影安插进 KV**。在 prompt 里点名 `double-register via reflection, a shot that lives twice—once in reality, once mirrored in adjacent glass or wet ground`。 |
| 双车规整语法（Dual-Hero Composition Grammar，跨品牌可迁移） | 从数据集 F 12 张行业标杆中归纳，**品牌级双车正典 = 单帧 + 完整 + 规整**，不是拆张也不是靠遮挡取巧。五种经典摆位：**DC-1 Mirrored 3/4 Pair**（前三四分 + 后三四分，3–6m 间距，微微相背，最常用），**DC-2 Parallel Stagger**（同朝向错位），**DC-3 Opposing Bookends**（镜像对称），**DC-4 Symmetric Line-up**（横排并列，适配 2–4 台），**DC-5 Close-Stage Pair**（紧凑舞台）。硬指标：两车完整入镜 + 同一地面 + 同一光向 + 同尺度 + 色彩一冷一暖或一明一暗 + 眼平 35mm + 水平规整。详见 SKILL.md `Multi-Subject Route`。 |

**反样本（跨题材可复用）**  
`plastic skin`（人像时）、`waxy HDR glow`、`teal-orange blockbuster grade`（除非用户要）、`mushy blur on primary subject`（主体该清晰时）。

### 数据集 B — 汽车生活 × 自然/建筑（8 张参考，2026-04-20）

| 维度 | 观察 |
|------|------|
| 类型 | Automotive **lifestyle** + commercial；静奢、阳光感或空灵雾感。 |
| 光 | 主：**golden hour** 暖侧光、长影；辅：**晨雾体积光**、**阴天柔光**（无强烈硬反射）。 |

4：庭院家庭与狗、金时刻、石木玻璃别墅、生活叙事。  
5：山脊徒步群体、低角广角、金时刻、户外品牌感、自然肤色与胶片颗粒感。  
6：深灰 SUV、松林雾、体积光、地面松针。  
7：双灰 EV、松林、低角、前车虚化框景、阴天柔光。  
8：双色轿车、龟裂大地、红岩、户外沙发与人物、金时刻游牧静奢。

### 摄影师参照（用户点名时启用）：Frédéric Schlosser

> **使用边界**：仅当用户**明确点名**「施洛瑟式 / Schlosser / 像他那种 / 要更艺术广告感」，或当条需求匹配**极简几何 + 时装化 + 强叙事广告**时才调用；**默认不**把他的手法作为全局画风。地景（原始森林、雪山、荒野）、纪实生活、静物产品等任务，只取**底层可迁移共性**，**不**强行套 Dutch tilt、极简几何布景、时装前景。

用户明确欣赏 **Frédéric Schlosser**（弗雷德里克·施洛瑟）的汽车摄影：**构图大胆、故事性强**，在保留 **商业品牌感**（车型、材质、灯语可读）的同时，不向「平庸产品录」滑。以下为从他系作品中抽象、可写进 MJ 的 **手法**（**不**等同逐张抄某位客户 Campaign，也**不必**在每句 prompt 写摄影师全名——MJ 对他名的响应不稳定，优先用 **视觉语义** 翻译）。

| 手法 | 说明（可迁移词方向） |
|------|----------------------|
| 叙事 ≠ 情节说明白 | 用 **姿态、遮挡、反射、目光方向** 制造停顿；例如前景人物虚、车实；或 **侧窗/漆面反射里的第二次人像**。 |
| 机位与张力 | **极低广角**（贴地前轮、座舱脚坑仰视）、**明显 Dutch angle**、**高机位对角鸟瞰**（充电线、地面几何作引导线）；车内可用 **后排向前看** 的沉浸机位。 |
| 布景逻辑 | 常把环境 **极简成几何**：大块 **白/浅灰地面与墙面**、**深饱和天顶**、**锐利日光阴影**；或 **沙丘/盐漠式负空间**，把车做成 **雕塑**，削弱「在普通马路」感。 |
| 色盘块状 | **古铜/玫瑰金车体** 与 **钴蓝天空**、**黑白人物** 的 **互补块面对撞**；控制色域，避免彩虹杂色。 |
| 光的戏剧性 | **硬光切割**、**长影**、**地上一滩聚光**（舞台感）、**逆光金雾 + 轮廓**；**半脸阴影**、**星芒眩光** 作情绪（慎用过量）。 |
| 内饰与体验 | **面料缝线、织布、屏显** 的材质可读；**天窗引入天色**；极少数张走 **高概念霓虹/赛车** 或 **装置化巨物**，仅当用户明确要极端变体。 |
| 露营叙事 | **尾门开启向内看**：床垫、暖光小灯、冷暖对撞的「把卧室放进荒野」——与森林帐篷类生活片同源，机位不同。 |

**迭代提示**：用户说「要像施洛瑟那种」时，从表中选 **2–4 个点**写进英文 prompt，并保留 **一条清晰的产品优先级句**（如 `hero vehicle razor-sharp, unmistakable premium EV design`），避免艺术机位把车冲没。

### 数据集 C — 松林 × 旅行车 × 精致露营（17 张参考，2026-04-20）

### 数据集 E — Schlosser × Xiaomi Villa Campaign（22 张参考，2026-04-21，来源 fredericschlosser.de/xiaomi/）

**说明**：Schlosser 为小米 **SU7 Ultra（荧光酸黄 sedan）+ YU7（葡萄紫 SUV）** 拍的**双车同系列** villa campaign。这是**当前最接近你想要的「双车同源 × 品牌资产化影像」范本**，比 D 更成熟地演示了**镜头分工 × 色彩呼应 × 系列统一**三件事。**慎用**：整体色调偏**高饱和艺术例外**（黄 + 紫互补），**只在用户要「强品牌色 / 舞台化 KV」** 时调用；日常写实路跑/生活场景仍回到 A/B/C。

**配色与场地策略**

| 要素 | 观察 |
|------|------|
| 两台车配色 | SU7 = **流荧光酸黄**（lime-yellow / chartreuse streak），YU7 = **深葡萄紫金属** + **玫瑰金/古铜轮毂** + 荧光磁/绿卡钳 |
| 色地桥接 | **一整块薰衣草紫绒毯 / 染色地坪** 横跨两车的多数场景，把两台不同色车**收进同一紫色宇宙**；这是此项目**系列统一**的关键机制 |
| 场地 | 热带现代主义别墅：玻璃幕、反射水池、**木百叶 / 金属窄条檐板**、椰影、远山海景；场地本身建筑语言统一（Hainan / Bali / 张家渡式的当代 resort） |

**镜头分工矩阵（这组 22 张覆盖了这 7 个职位——强烈建议我们以后做双车项目复用）**

| 职位 | 本项目示例帧 | 说明 |
|------|--------------|------|
| Hero golden-hour 逆光 | 01 / 05 / 06 / 08 | 穿百叶的硬光 + sunstar flare，车正脸或侧身 |
| Fashion 前景人肉占画 | 04 / 13 / 24 / 32 | 模特**贴镜大面积虚焦**，车子透过人的躯干被看见 |
| Bird's-eye 俯拍水景 | 07 | 高机位看车 + 水池反射 + 桌椅栈道 |
| Blue-hour 灯签名叙事 | 14 / 17 / 27 / 28 / 29 | 天色深蓝，**只靠尾灯 / DRL 发光**做画面主图形 |
| Light-signature 近摄 | 27（DRL）/ 14（尾灯环） | 「灯即 logo」，车其它部分暗退 |
| Interior POV / HMI | 02 / 21 / 26 | 驾驶位、HMI 屏（屏上壁纸=同一辆车，自我引用）、座舱仰拍 |
| 材质 / 细节 macro | 09（轮毂卡钳）/ 12（座椅针脚）/ 13（轮毂 + 人腿）/ 19（地面 projection logo） | 产品可读性与品牌仪式感 |

**光的两条注册（本项目共存）**

- **注册 E-1 百叶逆光金时刻**：木/金属窄条把太阳切成**硬光条**，车漆上出现 **长星芒 sunstar** 与 **棱镜彩虹闪色**（尤其 32 是明显的彩虹 lens flare）。常伴**玻璃反射复写**（5/23/25）。  
- **注册 E-2 蓝调 / 夜场图形灯带**：环境几近全黑，**灯带 = 主角图形**；模特被灯色**半脸 rim-lit**。不是「cyberpunk 赛博」，是**剧场化最小光**。

**人物策略**

- 亚裔女模为主，**服装雕塑化**：长外套 + 直筒裤 / 真丝套衫 / 皮裙 + 长靴；偶见男性西装。  
- **永远不笑**、**永远部分入画**（大腿、背影、侧脸、虚焦上半身）；角色作**尺度 + 叙事停顿 + 色彩呼应**。  
- **肤色自然**，不磨皮；**服装色**与车色或互补（紫 vs 黄）或同家族（紫衣配紫车）。

**Prompt 迭代时怎么用 E 数据集**

用户若说「想做双车同系列 / 品牌 KV 套图 / 类似小米 SU7 × YU7 那种」，建议按这个骨架写：
1. 先定**色盘三要素**：车色 A / 车色 B / **色地**  
2. 再定**场地建筑语言**（百叶檐 + 玻璃 + 水池 + 椰影，或你替换为其它统一地域语言）  
3. 按上面 **7 职位矩阵** 规划 5–8 张镜头分工，**每张一职**，**光的注册** 每组只选 1 条为主 + 1 条为辅  
4. 高饱和时**用大块暗场/负空间压制**，不要全画面霓虹

**反审美对齐校验**：注意这组虽高饱和，但**没有**：赛博霓虹堆砌、畸形广角、贴地仰视、风景压车、元素堆砌、无镜头分工——**全部通过**你的反审美清单。这说明「高饱和」与「廉价热闹」不是同义词，关键在**暗场与留白压制**。

### 数据集 F — Dual-Car Composition Benchmarks（12 张行业双车正典，2026-04-21）

**来源**：用户 2026-04-21 提供的 12 张行业品牌双车 campaign 参考（阿尔法·罗密欧 trio / 日产 IMk×Ariya / 广汽 / 华为 Stelato S9 × 2 组 / 保时捷 911 多组 / 保时捷 Taycan × Macan / 阿尔宾娜 BMW / 问界 Maextro S800 / 保时捷 911 Targa / Porsche 20 Years in China / Porsche 911 Turbo twin）。已归档 `reference-library/processed/2026-04-21_dual-car-compositions/`。

**说明**：这批是**行业对双车 campaign 的标准答案**，不是用户个人审美偏好——它定义了「双车完整展示」的**构图语法**本身。用户要求双车默认按这套写。

**五种经典双车摆位（均满足「完整入镜 + 规整摆位」）**：

| 代号 | 名称 | 摆位核心 | 本批代表 | 写法方向 |
|------|------|---------|----------|----------|
| **DC-1** | Mirrored 3/4 Pair | 车 A 前三四分 + 车 B 后三四分，朝向微微相背，间距 3–6m，形成微妙 yin-yang / V 形 | 04 Stelato 海港、08 Alpina 山脊、09 Porsche 911 沙漠、12 Porsche 20y 上海桥 | `one facing camera at a three-quarter front angle, the other presenting a three-quarter rear angle, softly mirrored away from each other with a clear 4–5 meter gap` |
| **DC-2** | Parallel Stagger | 两车同朝向，一前一后错开 10–30°，做纵深 | 02 Nissan、07 Taycan-Macan 红光、10 Maextro 外滩 | `both vehicles oriented in the same general direction but staggered in depth, the nearer car slightly more frontal and the further car receding at a ten-degree yaw offset` |
| **DC-3** | Opposing Bookends | 严格镜像对称，共享中央轴，朝内或朝外 | 05 Stelato 沙漠、11 Porsche 911 Targa 玻璃曲面 | `strictly symmetric bookend placement, the two cars mirroring each other across the frame's central axis, equal distance from center, identical camera-to-car distance` |
| **DC-4** | Symmetric Line-up | 并排同向一排，3/4 机位看过去，2–4 台通用 | 01 Alfa Romeo trio、03 GAC 双车 | `a formal horizontal line-up of the vehicles, all facing camera at a consistent three-quarter angle, evenly spaced` |
| **DC-5** | Close-Stage Pair | 两车紧凑 + 强环境框景（工作室 / 建筑结构） | 06 Porsche 911 twin 蓝墙竖幅、07 Taycan-Macan 红光 | `the two cars placed close together within an enclosing architectural or studio-stage frame, the environment tightly wrapping around them as a graphic backdrop` |

**全数据集共性（12 张全部命中）**：

- **两车 100% 完整入镜**，无裁切、无前景人物/道具遮挡
- **同一地面层**：沥青 / 打磨混凝土 / 盐碱地 / 石灰岩铺装 / 亮光工作室地面——**一个 ground plane 把两车锁在同一空间**
- **同一主光方向**：阴影方向完全一致；金时刻 / 蓝调 / 阴天柔光 / 棚灯聚光都只取 1 个主光
- **同尺度**：两车到相机距离相近（景深差 < 20%）
- **规整摆位**：间距 3–8m，从不贴合，从不叠影
- **车色对比不同家族**：银 + 红（03 / 04）、黑 + 香槟（10）、蓝 + 金（12）、白 + 黑（07）、蓝 + 黑（06 / 11）、香槟 + 樱桃红（05）——**杜绝双胞胎银/双胞胎黑**
- **眼平 / 略低机位 35mm**：地平线压在下三分之一（户外）或上三分之一（地面主导时）；没有一张贴地仰视、没有一张俯视
- **宽画幅**：11/12 张为横宽 16:9–21:9；唯一竖版是 06（品牌海报）
- **负空间给文案 / 品牌 logo 留位**（03 / 05 / 10 可见品牌字样；其余保留留白区）

**拍摄分工侧重**（与数据集 E 呼应）：
- **品牌 KV 正典**：DC-1 或 DC-3 + 金时刻 / 蓝调 → 04、08、09、11、12
- **建筑舞台化**：DC-5 或 DC-2 + 人工棚 → 02、06、07
- **地景叙事同框**：DC-1 / DC-4 + 真实城市/山/海 → 01、10、12
- **色地桥接**：03 的曲拱 + 蜜色石灰岩、05 的混凝土 + 沙丘色调、07 的纯红工作室——**数据集 E 的「地面色统一法」在双车里转化为「同一色地锁住两台不同色车」**

**迭代建议（给 Agent）**：
1. 双车任务进来先判题材 → 挑 DC 代号（户外地景默认 DC-1；KV 海报 DC-3；棚内 DC-5）
2. 再定**车色对比**（一冷一暖 / 一明一暗），与场景主色形成互补或呼应
3. 写法按 SKILL.md `Multi-Subject Route` 的英文骨架填空，**8 条硬指标全中**
4. 防翻车尾巴直接 copy

**反审美对齐校验**：12 张全部通过反审美清单——没有一张假 CG / 廉价赛博 / 贴地仰视 / 风景压车 / 前景杂物。**规整是双车的底线**。
